Core Rule

All tools run in the same browser window so localStorage data passes between them. Always open tools from the toolkit index or from the Dashboard's Tools tab — do not open multiple browser profiles at the same time or data will not transfer.

Structure
Three-Company Structure

Every deal moves through up to three companies in sequence. Each company has its own tools, P&L, and waterfall — but they all share one deal record in Firestore.

Company 1
Peach to Palms Property Group P2P

Role: Acquisitions. Finds the deal, negotiates the contract, decides the exit strategy.

Exit strategies: Wholesale (assignment fee), creative finance (subject-to, seller finance, novation), or flip pipeline (hand to Ironwood).

Key tools: MAO Calculator · BRRRR Analyzer · Deal Memo · Private Money Pitch (Lender Package) · P2P P&L · Track Record · Deal Pipeline

Who runs it: Ty Jackson. P2P is Ty's acquisitions operation — Jeff is not involved at the P2P level.

Waterfall: Revenue in → Opex + deal costs → 25% reserve → 10% portfolio fund → $2K/mo Ty draw → remainder forward.

Company 2
Anchor Pro Solutions AP

Role: Rehab execution. Scopes, bids, and manages the renovation on every deal that needs work.

Activates when: A P2P deal goes Under Contract and needs a scope of work. Also takes direct homeowner renovation clients.

Key tools: Remodel Estimator · Scope of Work · Room Scanner · Change Order Form · Draw Schedule · Invoice · AP P&L

Who runs it: Ty Jackson owns Anchor Pro. Ironwood covers sub labor and materials management.

Waterfall: Revenue in → Opex + job costs → 25% reserve → 10% reinvestment → $2K/mo Ty draw → remainder forward.

Company 3
Ironwood Property Group LLC Ironwood

Role: Asset ownership and disposition. Holds the renovated property, lists it, manages rentals, or executes BRRRR refinance.

Also does: Direct homeowner renovation contracts (full-scope home services, $400K+ Marietta/Cobb target market).

Key tools: Connected Deal Dashboard · Profit Waterfall · Ironwood Monthly P&L · Capital Engine · Client Portal · Rental Dashboard · New Construction Pipeline

Who runs it: Jeff (35% equity) and Ty (30% equity). Ironwood entity itself holds 35%.

Waterfall: Gross profit → P2P assignment fee off the top → remainder split 35% Jeff / 30% Ty / 35% Ironwood reinvestment.

Core Tool
The Hub — Connected Deal Dashboard

The Dashboard is the only tool that writes to Firestore in real time. Every other tool reads from or writes back to it via localStorage. Think of it as the deal's permanent file — open it first, close it last.

Tab 1
Overview Tab

Create the deal here. Set the address, ARV, asking price, exit strategy, and lead source. This creates the Firestore record every other tab and tool will update.

  • Fill ARV → MAO hint auto-calculates (70% × ARV)
  • Set exit strategy (Wholesale / Flip / BRRRR / Creative Finance / New Build)
  • Advance stages: Lead → Offer Sent → Under Contract → Scoped → Decision → Active → Closed / Dead
Open Dashboard
Tab 2
P2P Tab — Acquisitions Layer

Used by Peach to Palms. Record seller motivation, asking price, timeline, assignment fee, and deal costs. These numbers feed the Waterfall tab.

Tab 3
Anchor Pro Tab — Scope Layer

Used by Anchor Pro. Add scope line items (rooms + costs), set the scope total. This number syncs to the Waterfall as the rehab cost deduction.

Tab 4
Ironwood Tab — Asset Layer

Used by Ironwood LLC. Set sale price, list date, draw schedule, and final carrying/closing costs. Sale price flows into the Waterfall as gross revenue.

Tab 5
Waterfall Tab — Profit Splits

Auto-calculates from the other tabs. Shows gross profit, P2P assignment fee deduction, and final 35/30/35 split. No manual entry needed — just verify the numbers match reality.

Tab 6
Tools Tab — Push & Pull

The bridge between the Dashboard and every other tool.

  • Pull: Import rehab cost from Remodel Estimator, import BRRRR result, import MAO result
  • Push: Send this deal to BRRRR Analyzer, MAO Calculator, Remodel Estimator, Deal Memo, Private Money Pitch, or Track Record
  • P&L links: Jump directly to P2P P&L, Anchor Pro P&L, Ironwood P&L, or Profit Waterfall

Deal Flow — Step 1
Lead Comes In
Peach to Palms
1.1
Inbound Lead Sources

Leads enter through one of these channels:

  • REI Reply — AI voicemail answers inbound calls, scores the lead HOT / WARM / COLD, writes to Firestore, sends SMS alert
  • Home Valuation Tool — seller enters property info, gets instant value range, submits contact info → GHL pipeline
  • Seller Lead Form — direct intake for motivated sellers
  • Investor Inquiry Form — buyers and accredited investors opt in
  • Direct (cold call, DM, referral) — Ty enters manually into Dashboard
Home Valuation Tool Seller Lead Form
1.2
Create the Deal in the Dashboard

Open the Connected Deal Dashboard. Click + New Deal. Fill in:

  • Property address
  • Lead source (REI Reply / Cold Call / Direct Mail / Referral / Website / Inbound / Other)
  • Exit strategy (set a best guess — can change later)
  • ARV (from Zillow, PropStream, or comp pull)
  • Asking price (seller's number)

Click Save. The deal is now in Firestore and visible across all company tabs.

Open Dashboard
1.3
Advance Stage to "Offer Sent"

Once you've spoken to the seller and submitted an offer, click Advance to Offer Sent in the Overview tab. This timestamps the stage change in Firestore and keeps the pipeline accurate.

Deal Flow — Step 2
Qualify the Deal
Peach to Palms
2.1
Run the MAO Calculator

From the Dashboard Tools tab, click Open in MAO Calculator. The deal data (ARV, rehab estimate) loads automatically.

The MAO Calculator shows:

  • Maximum Allowable Offer = ARV × 70% − Rehab
  • Discount percentage from asking price
  • Whether the deal is worth pursuing (verdict label)
  • Rough profit split preview

When done, return to the Dashboard → Tools tab → Pull MAO Result to save it to the deal record.

MAO Calculator
2.2
Run the BRRRR Analyzer (if hold strategy)

If the exit strategy is BRRRR or rental hold, push to the BRRRR Analyzer from the Dashboard. It loads ARV, rehab, and purchase price automatically.

The BRRRR Analyzer shows:

  • All-in cost, refi loan amount, cash left in deal
  • Monthly cash flow and cap rate
  • Deal grade A–D and deal score
  • Whether you can pull your full investment out at refi

Pull the BRRRR result back to the Dashboard when done.

BRRRR Analyzer
2.3
Decision: Go / No-Go / Counter

Review the MAO (and BRRRR if applicable). Three outcomes:

  • Go — asking price is at or below MAO. Advance stage to Under Contract.
  • Counter — seller is above MAO but motivated. Submit MAO as counter offer.
  • No-Go — numbers don't work. Mark deal Dead in the Dashboard. Record note on why for the track record.

Deal Flow — Step 3
Run the Full Numbers
Peach to PalmsAnchor Pro
3.1
Walk the Property — Room Scanner

At the property, open the Room Scanner on your phone. Take photos room by room. The scanner:

  • Uses AI to identify distress signals and needed repairs
  • Voice dictation — narrate what you see, it types it
  • Builds a line-item estimate per room
  • Saves job history for future reference

The Room Scanner output becomes your baseline for the Remodel Estimator scope.

Room Scanner
3.2
Build the Scope — Remodel Estimator

Open the Remodel Estimator from the Dashboard (Tools tab → Open in Remodel Estimator). Data auto-populates.

In the Estimator:

  • Select company (Anchor Pro or Ironwood)
  • Add line items by room (labor + materials, markup %)
  • Assign paint colors — pick from 130+ Sherwin-Williams, Behr, Valspar, and Glidden colors with swatches
  • Select scope category tags (Kitchen / Bath / Flooring / Exterior, etc.)
  • Review the proposal preview and profit margin breakdown

When complete, click Push Rehab Cost to Dashboard. Return to the Dashboard → Tools tab → Pull Rehab Cost to lock it into the deal record.

Remodel Estimator
3.3
Advance Stage to "Scoped"

Once rehab cost is confirmed and pulled into the Dashboard, advance the deal stage to Scoped. This triggers the Anchor Pro layer — the scope is locked and ready for the bid.

Scope of Work & Proposal
Anchor Pro
4.1
Generate the Proposal

In the Remodel Estimator, click Build Proposal. The printed proposal includes:

  • Line items by category with labor + materials breakdown
  • Paint color swatches with brand, name, and code for each surface
  • Scope summary and total
  • Company branding (Anchor Pro or Ironwood depending on client type)

Save to PDF or print directly. This is the document you present to the homeowner client or attach to the deal file.

Remodel Estimator
4.2

Once scope is approved, generate the service agreement. Fill in client name, property, scope summary, total contract amount, and payment schedule (typically 3 phases: deposit / mid-project / completion).

Service Agreement
4.3
Set Up the Draw Schedule

For lender-funded deals, open the Draw Schedule tool. Group scope items into draw bundles (typically 3–5 draws). Each draw request goes to the lender with the bundle total and supporting scope items. The tool syncs draw totals back to the Deal Pipeline.

Draw Schedule
4.4
Scope of Work Tool (Full Build Tracker)

For larger projects, use the Scope of Work tool alongside the Estimator. It provides a full construction budget tracker with a client-facing view vs. internal view, and tracks build progress phase by phase.

Scope of Work

Deal Flow — Step 5
Present the Deal
Peach to Palms
5.1
Build the Deal Memo

From the Dashboard Tools tab, click Send to Deal Memo (Final). All deal numbers — ARV, rehab, purchase, assignment fee, closing costs, holding costs — load automatically.

In the Deal Memo, complete the 5 Deal Story sections:

  • The Property — beds/baths, sqft, year built, condition
  • The Location — neighborhood, proximity to schools/jobs/highways
  • The Comps — 2–3 recent comparable sales that support the ARV
  • The Seller Situation — motivation, timeline, flexibility
  • Why It's a Deal — the investment thesis in 2–3 sentences

The memo auto-calculates:

  • All-in cost (purchase + rehab + closing + holding)
  • Buyer's projected sell price at 85% ARV
  • Buyer's net profit and ROI %
  • Your assignment fee

Click Copy Text to copy a formatted version for text/email blast to your cash buyer list. Click Build Memo to generate the PDF-ready version.

Deal Memo
5.2
Investor Deal Summary (for equity partners)

For deals requiring equity partners or JV investors, use the Investor Deal Summary instead of or alongside the Deal Memo. It presents projected returns in an investor-facing format with IRR and equity split scenarios.

Investor Deal Summary
5.3
Advance Stage to "Decision"

Once the deal is presented to buyers or partners, advance the stage to Decision. You're waiting for a buyer to commit or an investor to fund. Follow up within 48 hours.

The Track Record tool reads the deal from localStorage and writes it to Firestore with net profit, rehab cost, and close date. It appears in Track Record instantly.

Technical Reference
Data Keys — How Tools Talk to Each Other

These are the localStorage keys that pass data between tools. You do not need to manage these manually — they are set and read automatically when you use the Push / Pull buttons. This reference is for troubleshooting if data is not auto-populating.

iw_dashboard_push
Set by Remodel Estimator → Push Rehab Cost. Read by Dashboard → Pull Rehab Cost.
iw_remodeler_output
Written by Remodel Estimator after push. Stores rehab total, scope summary, address.
iw_mao_input
Set by Dashboard → Push to MAO Calculator. Read by MAO on load.
iw_mao_output
Written by MAO Calculator after calc. Read by Dashboard → Pull MAO Result.
iw_brrrr_input
Set by Dashboard → Push to BRRRR Analyzer. Read by BRRRR on load.
iw_brrrr_output
Written by BRRRR Analyzer after calc. Read by Dashboard → Pull BRRRR Result.
iw_dealmemo_input
Set by Dashboard → Send to Deal Memo. Read by Deal Memo on load.
iw_dealmemo_output
Written by Deal Memo after calc. Stores buyer profit, ROI, assignment fee.
iw_pitch_input
Set by Dashboard → Send to Private Money Pitch. Read by Lender Package on load.
iw_trackrecord_input
Set by Dashboard → Log to Track Record. Read by Track Record on load — writes deal closed to Firestore.
iw_active_deal
Legacy key — written alongside tool-specific keys for backward compatibility with older tools.
Firestore: deals collection
Permanent deal records. Written by Dashboard (save) and Track Record (close). Read by all P&L tools on Import Deals.
Troubleshooting — Data Not Auto-Populating
  • Make sure you opened the destination tool after clicking Push in the source tool — the key must be set before the tool loads
  • All tools must be in the same browser (same localStorage context) — different browsers or incognito windows will not share data
  • If a tool shows a green import banner at the top, data loaded correctly — dismiss it to continue
  • If no banner appears and fields are blank, go back to the Dashboard → Tools tab and push again
